[Collection : 자료구조]

- 배열과 비슷한 구조
- 다수의 데이터를 쉽게 처리할 수 있는 메소드(Method)를 제공하는 API
- 다양한 자료구조 형이 존재
  
※ 다양한 자료구조형이 존재하는 이유 : 데이터의 성질에 따라 데이터를 관리하는 방식이 다르기 때문

[ArrayList]
- 배열의 단점을 보완한 자료구조형
- 배열과 달리 객체생성 시, 크기를 지정할 필요가 없으며, List의 크기는 유동적이다.
-인덱스고 존재하며, 중복데이터 저장이 가능함.

[HashMap]
- 인덱스 대신 키 값으로 데이터를 액세스한다.
- List와 달리 인덱스가 없으며, 키와 값만 존재함.
- 값을 관리하고자 할때 키를 이용해 관리할 수 있다.

[HashSet]
- 데이터의 순서가 없다. 중복데이터 저장이 불가하다.

[Iterator : 반복자]
- 반복적으로 검색할때 아주 유용한 인터페이스

[TreeSet]
- 트리 구조로 저장되며, 좌측은 작은 값, 우측은 큰 값이 저장된다.
